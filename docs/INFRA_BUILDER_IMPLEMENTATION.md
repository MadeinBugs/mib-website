   - Headers: `Authorization: Bearer <N8N_QUOTE_WEBHOOK_SECRET>`

3. Add Webhook trigger node. Configure it to validate the `Authorization` header against `N8N_QUOTE_WEBHOOK_SECRET` (stored as n8n credential).

4. Add Code node to parse the Supabase row and build payloads for Twenty (Company, Person, Opportunity, Task). Extract domain from `studio_website` for Company matching.

5. Configure Twenty API credentials in n8n (store `TWENTY_API_KEY`, `TWENTY_BASE_URL`).

6. Add HTTP Request nodes per Section 12.1, steps 3–6, with upsert logic using IF nodes to branch between "found → PATCH" and "not found → POST":
   - **Company upsert:** Match by domain (extracted from `studio_website`) OR name. If found, PATCH; else POST.
   - **Person upsert:** Match by email. If found, PATCH with updated `companyId`; else POST.
   - **Opportunity idempotency for retries:** Before creating an Opportunity, check if one already exists with custom field `external_id = <quote_uuid>`. If found, PATCH instead of POST. This prevents duplicate Opportunities when the admin triggers "Retry Twenty sync" after a partial failure.
   - **Task:** Always create new. Due date: `now + 2 business days`. Assignee: `TWENTY_ASSIGNEE_USER_ID` (your user ID).

7. Add HTTP Request node to PATCH the Supabase `quote_requests` row with `twenty_opportunity_id`. Uses `SUPABASE_SERVICE_ROLE_KEY` configured in n8n.

8. Add HTTP Request node to post to `DISCORD_QUOTE_SYNC_WEBHOOK_URL` with success embed linking to the Twenty Opportunity:
   ```
   ✅ Twenty synced for quote ${uuid}
   Opportunity: ${opportunityName}
   Company: ${companyName}
   Amount: ${formattedAmount}
   Task due: ${taskDueDate}
   ```

9. Add error branch: if any node fails, post to `DISCORD_QUOTE_SYNC_WEBHOOK_URL`:
   ```
   ⚠ Twenty sync failed for quote ${uuid}
   Failed step: ${nodeName}
   Error: ${error}
   Manual retry: click "Retry Twenty sync" on /admin/quotes for this quote
   ```
   Do not auto-retry; admin view provides manual recovery.
